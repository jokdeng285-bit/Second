# Spark — 40 Commit Build Scripts **Tested end-to-end. Produces exactly 40 commits, in order, with no errors.** ## What's in this folder | File | Commits | What it builds | |------|---------|-----------------| | `phase1-setup.sh` | #01–#07 | Project init, README, .gitignore, folders, CDN links, fonts, docs | | `phase2-html.sh` | #08–#15 | All 8 HTML pages (index, how-it-works, browse, pricing, success-stories, blog, about, contact) | | `phase3-css.sh` | #16–#24 | Full stylesheet — tokens, navbar, buttons, cards, pricing, FAQ/forms, footer, dark/light mode | | `phase4-javascript.sh` | #25–#32 | Particles, scroll bar, cursor glow, mobile nav, theme toggle, scroll reveal, FAQ accordion, profile filter | | `phase5-features.sh` | #33–#36 | Billing toggle, signup modal, contact validation, like buttons | | `phase6-polish.sh` | #37–#40 | Mesh background, **"Crafted by Jok"** footer credit, mobile fixes, final README | | `run-all-phases.sh` | — | Runs all 6 scripts above in order, back to back | ## How to use it ### Option A — run everything at once ```bash mkdir spark && cd spark # copy all 7 .sh files into this folder chmod +x *.sh ./run-all-phases.sh ``` This creates 40 commits locally in one go. ### Option B — run phase by phase (recommended if you want to inspect each step) ```bash mkdir spark && cd spark # copy all 7 .sh files into this folder chmod +x *.sh bash phase1-setup.sh # → 7 commits bash phase2-html.sh # → 8 commits (15 total) bash phase3-css.sh # → 9 commits (24 total) bash phase4-javascript.sh # → 8 commits (32 total) bash phase5-features.sh # → 4 commits (36 total) bash phase6-polish.sh # → 4 commits (40 total) ``` Run `git log --oneline | wc -l` after any phase to confirm the count. ## Pushing to GitHub 1. Create a **new, empty** repository on GitHub (don't tick "Add a README" — your script already creates one) 2. Copy the repo URL 3. Run: ```bash git remote add origin https://github.com/YOUR_USERNAME/spark.git git branch -M main git push -u origin main ``` You'll see all 40 commits appear in your GitHub history with correct timestamps and messages. ## Important notes - **Git identity required.** If you haven't set one up on this machine: ```bash git config --global user.name "Jok" git config --global user.email "you@example.com" ``` - **Run the phases in order.** Phase 3 edits the same `css/style.css` file that Phase 1 creates; Phase 4–5 all append to the same `js/main.js`. Skipping a phase will break later ones. - **Don't run a phase twice in the same folder.** Each script assumes it's running fresh after the previous phase. If something goes wrong, delete the folder and start over from Phase 1. - **The `.sh` scripts themselves are gitignored** — they won't show up as files in your repo, only the actual site files will. - Every phase script has `set -e`, so if any step fails, the whole script stops immediately rather than silently continuing. ## Verifying the result ```bash git log --oneline | wc -l # should print 40 git ls-files # lists exactly: html pages, css/style.css, js/main.js, README.md, CONTRIBUTING.md, .gitignore, _base-template.html grep -l "creator-tag" *.html # should list all 8 html pages ``` ## Final file structure produced ```
